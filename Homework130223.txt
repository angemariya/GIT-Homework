1. Предварительно нужно зайти в свою учетную запись на GitHub.com, найти там репозиторий который необходимо склонировать и открыть его. Далее кликнуть по кнопке зеленого цвета <Code>, скопировать SSH-ключ. В терминале (или консоли GitBash) на компьютере необходимо открыть ту директорию (или создать новую) , в которую необходимо скопировать репозиторий. После этого в терминале вводим следующую команду: git clone и вставить скопированный ключ SSH. 

2.Для того, что отправить локальную папку с компьютера на GitHub.com нужно:
1) Создаем новый пустой репозиторий на Github.com и скопируем ключ SSH. 
2) Нужно зайти через консоль в ту папку, которую необходимо опубликовать на гитхабе. 
3) В консоле ввести команду инициализации локального репозитория „git init“
4) Вводим команду „git status“, для просмотра файлов которые еще не отслеживаются.
5) Добавляем файлы необходимые для публикации через команду „git add “ и имена файлов и их расширения. Или „git add .“ чтобы добавить все файлы. 
6) Добавляем коммит: „git commit -m ‚название коммита’ “ - название коммита должно быть осознанным, как правило по действиям, которые мы произвели во время работы с файлами. 
7) Создаем и переключаемся на главную ветку main: „git branch -M main“
8) Привязываем локальный репозиторий к удаленному: git remote add origin <скопированный ключ SSH>
9)Отправляем файлы в удаленный репозиторий: git push -u origin main

3. Чтобы получить изменения с github.com на локальный репозиторий необходима команда git pull

4. Git status - проверка состояния файлов (есть изменения в файловой системе локального репозитория или нет)

Git remote -v просмотр подключенных удаленных репозиториев и протокол их подключения
