1.    git checkout <имя файла> отменить все последние изменения в 
подконтрольном 
файле
2.     git restore —stage <имя файла> - возврат файла из staged area в 
working 
directory
3.    git revert HEAD - откатить изменения в последнем коммите 
4.    git checkout -b <имя ветки> создать ветку и переключиться на нее
5.    Fast-forward merge когда текущая ветка не имеет дополнительных 
коммитов по сравнению с веткой, которую мы мержим. Если мы зафиксировали 
изменения в текущей ветке, которых нет в ветке, которую мы хотим 
объединить, git выполнит объединение без fast-forward merge. При попытке 
объединить ветки, Git покажет где происходит конфликт. Мы можем вручную 
удалить изменения, которые не хотим сохранять, сохранить изменения, снова 
добавить файл и закоммитить изменения.
6.    git branch -d <имя ветки>
